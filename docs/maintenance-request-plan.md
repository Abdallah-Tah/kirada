# Maintenance Request Plan

## Current Build Slice

- [x] Tenant request form with category, location, access notes, urgency guidance, and photos.
- [x] Private maintenance attachments with internal visibility controls.
- [x] Request detail timeline, comments, request photos, and resolution photos.
- [x] Email notifications for new requests, comments, and status updates.
- [x] Tenant-safe status workflow:
  - Tenant can cancel an open request.
  - Tenant cannot move their own request to in progress.
  - Tenant can confirm a resolved request by closing it.
  - Tenant can reopen a resolved request if the issue is not fixed.
- [x] Linked maintenance conversations from the request page.

## Next High-Value Features

- [ ] Landlord maintenance dashboard queue for urgent, unassigned, overdue, and recently updated requests.
- [ ] Tenant dashboard section for recent requests with status and quick request creation.
- [ ] Scheduled visit date, due date, and appointment window tracking.
- [ ] Vendor/contractor details, cost estimate, final cost, and repair notes.
- [ ] Activity log for created, assigned, status changed, photos added, and comment added events.
- [ ] In-app database notifications in addition to email.
- [ ] Attachment management: delete, replace, mark internal/public, show uploader/date.
- [ ] Better request index filters: category, property, unit, overdue, unassigned, has photos.
- [ ] Scope maintenance users by landlord/property instead of showing all maintenance users globally.

## Later Enhancements

- [ ] Emergency acknowledgement for urgent requests.
- [ ] Resolution checklist: cause, work completed, parts used, tenant notified.
- [ ] Mobile request cards for the maintenance index table.
- [ ] Optional request ratings/feedback after closure.
